# Examples

Loadable Frontier problems: combinatorial, multi-objective decisions beyond a spreadsheet. Each ships a `problem.json` (objectives, approach, constraints, scenarios), a `scores.json` (options, scores, interaction matrices), a pre-solved `solutions.json` (results restored on load), and a workflow walkthrough in its own README — one natural user ask per phase, with the tools that fire and the response shape to expect. Each is pre-framed and pre-scored, so its walkthrough picks up the [main workflow](../README.md#workflow) at **Solve** (steps 1–2, Frame and Score, are already done). Two examples also ship the **upstream kit** — a user-voiced `BRIEF.md` + raw `data.csv` whose framing reconstructs the canonical model verbatim, so you can start at Frame the way a real user would: [capital project selection](capital_project_selection_120/) and [scarce supply rationing](scarce_supply_rationing/).

**Solver-exact explainability:** the [budget allocation](budget_allocation/), [production mix](production_mix/), and [scarce supply rationing](scarce_supply_rationing/) examples (purely linear **LP**) and the [investment portfolio](investment_portfolio/), [supplier selection](supplier_selection/), and [generation capacity planning](capacity_planning/) examples (mean-variance **QP**) each run the full workflow – `solve run` → `solve solver="highs"` → `explore certify` → `explore sensitivity` – surfacing shadow prices ("where to invest") and reduced costs ("near-misses", and the price of a contractual floor) straight from the optimizer on the continuous (LP / QP) paths. (Binary MILP shapes have no duals and fall back to the frontier-inferred estimate.)

**Governance (audit):** on binary selection problems, `explore audit` proves a property across the **whole feasible space** or returns a counterexample witness. [Claims investigation triage](claims_investigation_triage/) is the flagship (a compound guarantee in one conjunctive call, with an emergent core the constraints never state); [research cohort selection](research_cohort_selection/) shows the emergent-tightening flavor; [charging network siting](charging_network_siting/) proves coverage commitments on a shape the exact solver *declines* to optimize — audit is a feasibility solve, so a quadratic objective doesn't block it.

**GPU (cuOpt):** the hosted beta is CPU-only, so the GPU backend ships as a ready-to-run Colab template — [cuopt_colab.ipynb](cuopt_colab.ipynb) runs the same explore → exact → certify → duals arc with `solver="cuopt"` on **any bundled example — or your own problem** (it scaffolds both paths): open it in Colab, switch the runtime to GPU, run top to bottom.

**Load by name:** with the engine running, `model load source="investment_portfolio"` rebuilds any example directly – scenarios, interaction matrices, and all – with no manual re-entry. Problems you build save back to this same format (into a gitignored `saved/` library) via `model save`. See [Saving & loading](../README.md#saving--loading).
